WalkWise

A campus navigation tool that helps Pitt students avoid construction detours, stay informed about closures, and find safe, efficient walking routes.

Target users: Pitt students who walk on campus Obstruction avoidance Real-time reports Community updates

Project Overview

Problem

With campus constantly under construction, students run into sudden sidewalk closures, blocked paths, and confusing detours. That makes it harder to get to class on time and navigate safely.

Solution

WalkWise lets students see active sidewalk closures, report new obstacles, and follow safer alternate routes around campus disruptions.

Key Features

  • Map-based view of construction and hazards around campus
  • Real-time obstruction reports from the student community
  • Alternative routes to avoid closures and unsafe paths
  • Basic social space for sharing updates and discussing routes

Target Audience

Our primary users are University of Pittsburgh students who:

  • Walk to classes, dorms, and campus buildings every day
  • Are affected by frequent construction and sidewalk closures
  • Want simple, quick navigation support without a heavy learning curve

The design decisions, tasks, and evaluation all focused on this group.

Prototypes

We moved from quick lo-fi sketches to hi-fi interactive prototypes. Each stage helped us refine navigation, information layout, and social features.

Lo-Fi Prototypes

Early paper and low-fidelity screens helped us decide what belonged on the home screen, how lane closures should appear, and whether social features were worth including.

Home Screen – simplified
Early home screen was cluttered with too many buttons. Later versions reduced options and made the main map more prominent.
Lane Closure – more context
Initial design only showed a closure marker. We later added more detail and an obvious button to view an alternate route.
Social Features – added later
The first version lacked social features. We introduced a basic community space so users could share updates and feel confident in the data.

Hi-Fi Prototypes

High-fidelity prototypes were used in usability testing. They helped us refine layers, information cards, and sign-in behavior based on user feedback.

Unified Map + Layers
Instead of separate maps for navigation and hazards, we added a layers button to toggle routes, hazards, or both, and highlighted safe routes versus active closures.
Construction Detail Card
Pins now open a detailed card with status, estimated completion dates, and a “learn more” option instead of only showing a location name.
Guest Mode
Usability testing showed some users didn’t want to create an account immediately, so we added a Continue as Guest option with limited but useful access.

Evaluation Summary

Methods

  • 5 Pitt students who regularly walk on campus
  • Hi-fi prototype evaluated with think-aloud usability tests
  • Tasks: create an account and navigate to a construction alert
  • Collected both task performance data and post-task SUS scores

Key Findings

  • Visuals and overall layout felt simple and approachable.
  • Users struggled to find specific construction notification pages.
  • Some screens lacked an obvious way to go back.
  • Several people wanted a guest mode instead of mandatory account creation.
  • Average SUS score was around 65, suggesting room for improvement.

Design Changes

  • Combined navigation and construction alerts into a single main interface.
  • Added a clear guest sign-in option with map viewing and basic use.
  • Improved navigation structure and feedback to reduce dead-ends.

These changes directly came from user testing sessions and short follow-up interviews.
